# Event Dashboard

ইভেন্ট ড্যাশবোর্ড (`acd.toolforge.org`) হলো একটি চমৎকার ও প্রিমিয়াম ওয়েব অ্যাপ্লিকেশন যা আউটরিচ ইভেন্ট বা কর্মশালার আয়োজকদের জন্য দ্রুত নতুন উইকিপিডিয়া অ্যাকাউন্ট তৈরি করতে সহায়তা করে।

এটিতে সাধারণ ব্যবহারকারীদের জন্য একটি সুন্দর নিবন্ধন আবেদন ফর্ম রয়েছে এবং অ্যাডমিনদের জন্য রয়েছে সম্পূর্ণ নিরাপদ ড্যাশবোর্ড ও সেটিংস ব্যবস্থা।

## বৈশিষ্ট্যসমূহ (Features)

- **বাংলা ইন্টারফেস (Bangla UI):** ব্যবহারকারী এবং অ্যাডমিন উভয়ের জন্যই সম্পূর্ণ ইন্টারফেস বাংলায় ডিজাইন করা হয়েছে।
- **রিয়েল-টাইম ব্যবহারকারী নাম ও অ্যান্টি-স্পুফ পরীক্ষা (Real-time checks):** ব্যবহারকারী যখন আবেদন ফর্মে তার নাম লিখবেন, তখন ব্যাকএন্ডের মাধ্যমে রিয়েল-টাইমে bnwiki এবং SUL (গ্লোবাল) নেটওয়ার্কে নামটি ইতিমধ্যে আছে কিনা এবং AntiSpoof extension অনুযায়ী কোনো সংঘাত (conflict) তৈরি করছে কিনা তা পরীক্ষা করা হয়।
- **নিরাপদ ও স্বয়ংক্রিয় অ্যাডমিন যাচাইকরণ (OAuth Login & Admin Check):** অ্যাডমিন প্যানেলে প্রবেশ করতে উইকিপিডিয়ার অফিসিয়াল OAuth 2.0 ব্যবহার করা হয়। লগ ইন করার পর সিস্টেম স্বয়ংক্রিয়ভাবে যাচাই করে ব্যবহারকারী `bd.wikimedia.org` অথবা `bn.wikipedia.org` এর প্রশাসক (sysop) কিনা।
- **স্বয়ংক্রিয় ডাইনামিক উইকি টার্গেট (Dynamic Account Creation Target):** ব্যবহারকারী যে উইকির প্রশাসক, অ্যাকাউন্ট তৈরির আবেদনটি অনুমোদন করার পর অ্যাকাউন্টটি সেই নির্দিষ্ট উইকিতেই তৈরি হবে:
  - অ্যাডমিন যদি `bd.wikimedia.org`-এর প্রশাসক হন, তবে অ্যাকাউন্টটি `bd.wikimedia.org`-এ তৈরি হবে।
  - অ্যাডমিন যদি `bn.wikipedia.org`-এর প্রশাসক হন, তবে অ্যাকাউন্টটি `bn.wikipedia.org` (bnwiki)-তে তৈরি হবে।
- **স্বয়ংক্রিয় অ্যাকাউন্ট তৈরি (Automated Account Creation):** অ্যাডমিন ড্যাশবোর্ডে "তৈরি করুন" বাটনে ক্লিক করলে সাময়িক পাসওয়ার্ড ইমেইলে পাঠিয়ে মিডিয়াউইকি API-এর মাধ্যমে স্বয়ংক্রিয়ভাবে অ্যাকাউন্ট তৈরি হয়ে যায়। এতে ইভেন্টের নামসহ লগ সামারি যুক্ত থাকে: `<ইভেন্টের নাম>-এর অংশগ্রহণকারীর জন্য অ্যাকাউন্ট তৈরি করা হলো।`
- **ইভেন্ট ও রেজিস্ট্রেশন সেটিংস (Event & Registration Control):** অ্যাডমিন সেটিংস থেকে চলমান ইভেন্টের নাম পরিবর্তন করতে এবং যেকোনো সময় নিবন্ধন প্রক্রিয়া শুরু বা বন্ধ করতে পারেন। নিবন্ধন বন্ধ থাকলে সাধারণ ব্যবহারকারীরা "বর্তমানে কোনো ইভেন্ট চলছে না" পৃষ্ঠাটি দেখতে পাবেন।

## প্রযুক্তিগত বিবরণ (Tech Stack)

- **ব্যাকএন্ড:** Node.js (Express)
- **ডাটাবেজ:** SQLite (ফাইল ভিত্তিক, কোনো বাহ্যিক ডাটাবেজ ইন্সটলেশনের প্রয়োজন নেই)
- **স্টাইলিং:** Tailwind CSS (ডিজাইন প্রোটোটাইপ অনুযায়ী প্রিমিয়াম ও রেসপন্সিভ ডিজাইন)

## প্রজেক্ট সেটআপ নির্দেশিকা (Setup Instructions)

### ১. ডিপেন্ডেন্সি ইনস্টলেশন
প্রজেক্টের মূল ডিরেক্টরিতে টার্মিনাল খুলে নিচের কমান্ডটি রান করুন:
```bash
npm install
```

### ২. কনফিগারেশন (.env)
প্রজেক্ট ডিরেক্টরিতে একটি `.env` ফাইল তৈরি করুন (অথবা `.env.example` ফাইলটি কপি করে `.env` নাম দিন) এবং নিচের মানগুলো দিন:
```env
PORT=3000
WIKIMEDIA_CLIENT_ID=your_client_id_here
WIKIMEDIA_CLIENT_SECRET=your_client_secret_here
WIKIMEDIA_REDIRECT_URI=http://localhost:3000/auth/callback
SESSION_SECRET=dev_session_secret_for_local_testing_123456
```

### ৩. অ্যাপ্লিকেশন রান করা
ডেভেলপমেন্ট মোডে (কোড পরিবর্তনের সাথে সাথে রিলোড হবে) রান করতে:
```bash
npm run dev
```

প্রোডাকশন মোডে রান করতে:
```bash
npm start
```

সার্ভার চালু হলে ব্রাউজারে `http://localhost:3000` এ প্রবেশ করুন।

## লোকাল টেস্টিং এবং মক অ্যাডমিন মোড (Local Testing / Mock OAuth)

উইকিমিডিয়া মেটাতে OAuth Consumer প্রস্তাব করার পূর্বে সম্পূর্ণ প্রজেক্টের কার্যকারিতা লোকালহোস্টে পরীক্ষা করার জন্য এই টুলে একটি **মক অ্যাডমিন মোড (Mock OAuth)** যুক্ত করা হয়েছে।

যদি `.env` ফাইলে `WIKIMEDIA_CLIENT_ID` কনফিগার করা না থাকে (বা placeholder থাকে), তবে লগ ইন পেজে ক্লিক করলে সিস্টেম আপনাকে স্বয়ংক্রিয়ভাবে একজন প্রশাসক হিসেবে অ্যাডমিন প্যানেলে লগ ইন করিয়ে দেবে।

- **বাংলা উইকিপিডিয়া (bnwiki) অ্যাডমিন হিসেবে পরীক্ষা করতে:**
  সরাসরি `http://localhost:3000/login` অথবা `http://localhost:3000/login?wiki=bn` লিংকে যান। এতে অ্যাকাউন্ট তৈরির টার্গেট সেট হবে `bn.wikipedia.org`।
- **উইকিমিডিয়া বাংলাদেশ (bdwikimedia) অ্যাডমিন হিসেবে পরীক্ষা করতে:**
  `http://localhost:3000/login?wiki=bd` লিংকে প্রবেশ করুন। এতে অ্যাকাউন্ট তৈরির টার্গেট সেট হবে `bd.wikimedia.org`।

## উইকিমিডিয়া OAuth 2.0 কনজিউমার প্রস্তাব করার নিয়ম

প্রোডাকশন ডোমেইনে (যেমন: `acd.toolforge.org`) এই টুলটি চালাতে চাইলে আপনাকে উইকিমিডিয়া মেটাতে একটি OAuth 2.0 Consumer প্রস্তাব করতে হবে:

1. উইকিপিডিয়ায় লগইন থাকা অবস্থায় [Special:OAuthConsumerRegistration](https://meta.wikimedia.org/wiki/Special:OAuthConsumerRegistration/propose/oauth2) লিংকে যান।
2. নতুন Consumer প্রস্তাব করার সময় নিচের তথ্যগুলো প্রদান করুন:
   - **Application Name:** `Wikimedia Bangladesh Outreach Toolkit`
   - **OAuth Version:** `OAuth 2.0`
   - **Callback URL:** `https://acd.toolforge.org/auth/callback` (অথবা আপনার কাস্টম ডোমেইন callback)
   - **Grants Required:**
     - *Basic rights* (ব্যবহারকারী সনাক্তকরণ এবং ইউজারনেম জানার জন্য)
     - *Create accounts* (অ্যাডমিনের পক্ষে অ্যাকাউন্ট তৈরি করার অনুমতি)
3. Consumer সাবমিট করার পর মেটা-উইকির অ্যাডমিনদের দ্বারা এটি অনুমোদিত হলে আপনি একটি **Client ID** (Consumer Key) এবং **Client Secret** (Consumer Secret) পাবেন।
4. প্রাপ্ত কি-গুলো আপনার ডোমেইনে চালিত টুলের `.env` ফাইলে বসিয়ে দিন।
